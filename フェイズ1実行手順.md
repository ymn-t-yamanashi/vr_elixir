# フェイズ1実行手順

## 1. 前提
- ホストで `mix` は実行しない。
- すべて Docker 内で実行する。
- Resonite 側で ResoniteLink を有効化し、`host.docker.internal:12512` で待ち受ける。

## 2. 品質ゲート
```bash
docker compose run --rm app bash -lc 'cd resonite_link_ex && mix local.hex --force && mix local.rebar --force && mix deps.get && mix format --check-formatted && mix compile --warnings-as-errors && mix check.docs && mix credo --strict && mix test --cover'
```

## 3. 代表命令再現（E2E 手順）
1. `requestSessionData` を送信し、成功応答を受信する。
2. `addSlot` で Slot を作成する。
3. `updateSlot` で Transform を更新する。
4. `addComponent` で Component を追加する。
5. `updateComponent` で Component 値を更新する。
6. `removeComponent` で Component を削除する。
7. `removeSlot` で Slot を削除する。

## 4. Quad 表示手順
1. `Scene.quad_plan("Root", "QuadA")` を生成する。
2. 返却される 4 命令（`addSlot` / `updateSlot` / `addComponent` / `updateComponent`）を順に送信する。
3. 各応答で `messageId` を照合し、失敗時は `error_reason` をログ確認する。

## 5. 再接続試験（3回）
1. 接続確立後に切断を発生させる（close frame か tcp error）。
2. `handle_disconnect/2` 経由で `reconnecting=true` になることを確認する。
3. 再接続後の `requestSessionData` 成功応答で `session_ready=true` かつ `reconnecting=false` へ戻ることを確認する。
4. 1〜3 を 3 回繰り返す。

## 6. 失敗ケース確認
- 不正 `$type` は `{:error, :invalid_request}`。
- 存在しない `slotId` / `componentId` は Resonite 側応答に従って `{:error, :not_found}` もしくは `{:error, {:remote_error, code, message}}` として扱う。
