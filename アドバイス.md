# アドバイス

最終確認日: 2026-05-04

## 1. ResoniteLinkのポート管理について

### 1-1. ResoniteLinkPlus（Mod）
ResoniteLinkPlus は実在するResonite向けModで、ResoniteLinkの運用を補助する機能があります

確認できた機能（Thunderstore掲載）:
- New WorldダイアログでResoniteLinkを有効化できる
- ポートを指定できる
- 指定開始ポートから自動インクリメントできる
- ワールド終了時のクリーンアップ機能

参照:
- https://thunderstore.io/c/resonite/p/hazre/ResoniteLinkPlus/
- https://github.com/hazre/ResoniteLinkPlus

## 2. ポート自動発見について

### 2-1. UDP 12512 でのポート告知
ResoniteLink公式リポジトリの実装に `ANNOUNCE_PORT = 12512` があり、リンクセッションの検出に使われています

参照（公式実装）:
- https://github.com/Yellow-Dog-Man/ResoniteLink/blob/master/ResoniteLink/LinkSessionListener.cs

## 3. 公式仕様として押さえる点

- ResoniteLink は公式に WebSocket + JSON プロトコル
- 仕様はbeta扱いで、破壊的変更の可能性あり
- 実装時は公式Modelsとドキュメントを一次情報にする

参照:
- https://github.com/Yellow-Dog-Man/ResoniteLink
- https://yellow-dog-man.github.io/ResoniteLink/

## 4. 補足（運用上の注意）

- AIエージェントに毎回ポート探索をさせる場合、ログ解析だけに依存させない方が安定する
- 可能なら以下の優先順で探索する
  1. ResoniteLinkの告知/検出機構（UDP 12512）
  2. 明示設定された固定ポート
  3. 最後の手段としてログ解析
