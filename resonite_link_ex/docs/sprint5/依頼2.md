# 依頼: ResoniteLinkEx.Core を使ったサンプルプログラム

## 目的
`ResoniteLinkEx.Core` は「`$type` 単位の低レベルAPIで、送信しやすい内部リクエスト（`%{type, payload}`）を作る」層なので、
これを使って「何をどう送れば Resonite 側が動くか」を最短で理解できるサンプルを用意する。

## 提案サンプル案（最小で動くやつ）
ファイル: `resonite_link_ex/examples/sprint5_core_sample.exs`

やること:
1. `ResoniteLinkEx.Client.start_link/0` で接続する
2. `Client.session_ready?/1` が `true` になるまで待つ（既存サンプルと同じ待機）
3. `Core.request_session_data/1` でリクエストを作り、`Client.send_json/2` で送ってレスポンスを表示する
4. `Core.add_slot/2` で Slot を追加する（例: `name: "CoreBox_#{System.system_time(:millisecond)}"`）
5. 追加した Slot の `slotId`（またはレスポンス内の id）を取り出せるなら取り出し、`Core.update_slot/2` で位置を動かす
6. 最後に `Core.remove_slot/2` で掃除する（作った Slot を消す）

ポイント:
- `call_core` は現状の実装だと「リクエスト生成のみ」で実際に送れていない可能性があるため、サンプルは `ResoniteLinkEx.Client.call/3` 経由で行う。
  - 前提: `ResoniteLinkEx.Core` は内部で `Client.call/3` を使う形に見直す（PID を渡せば送信まで進む）。
- 既存の `Shapes.spawn_cube/2` などの「高レベルAPI」と違い、`$type` と payload を自分で扱う練習になる。

## 追加の提案（任意、発展）
発展1: 「Coreだけで addSlot/updateSlot/removeSlot を直列に実行」する mini ワークフロー関数を作る
- `examples/` はスクリプトなので、まずは exs だけで完結させる

発展2: `dataModelOperationBatch` を使った一括送信サンプル
- Slot 追加 + Component 追加 + 位置更新 を1メッセージで送る（順序保証）
- これができると「Coreの旨味（薄いAPIを積み上げて操作を作る）」が伝わりやすい

## 想定実行方法
```bash
cd resonite_link_ex
mix run examples/sprint5_core_sample.exs
```

## 先に決めたいこと（実装に入る前の確認事項）
- `ResoniteLinkEx.Core` で実装出来ていない部分は、サンプル実装に入る前に必要分を実装する前提とする。
- `ResoniteLinkEx.Core` は `{:ok, transport} = Client.start_link()` のあとであれば使える前提とする。
- サンプルは必ず下記の初期化手順を使う前提とする。
```elixir
{:ok, transport} = Client.start_link()
ResoniteLinkEx.NameResolver.clear_resonite_link_ex_slot(transport)
ResoniteLinkEx.NameResolver.ensure_slot_id(transport, "ResoniteLinkEx")
```
- サンプルは「ワールドを汚さない」方針で、作った Slot は最後に削除する（削除に失敗したら名前にタイムスタンプを付けて見分けやすくする）。
- Slot の親は基本 `Root` 想定でよい（必要なら `requestSessionData` のレスポンスからルート参照を使う形に拡張する）。
