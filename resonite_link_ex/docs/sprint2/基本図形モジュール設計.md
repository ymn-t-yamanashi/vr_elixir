# 基本図形モジュール設計（スプリント2）

## 1. 目的
- `quad` `cube` `sphere` `cylinder` `capsule` `ring` `grid` を一貫したI/Fで生成できるようにし、図形生成ロジックを再利用可能にする。

## 2. 対象範囲
- 対象図形: `quad` `cube` `sphere` `cylinder` `capsule` `ring` `grid`
- 対象処理:
  - 図形生成用メッセージ（`addSlot` / `addComponent` / `updateComponent`）の組み立て
  - 実行順序の固定化
- 対象外:
  - 高度な材質設定（テクスチャ、複雑なPBR）

## 3. モジュール構成
- 追加モジュール: `ResoniteLinkEx.Shapes`
- 依存モジュール:
  - `ResoniteLinkEx.Transport`（送信）
  - `ResoniteLinkEx.Client`（接続状態確認）

## 4. 公開I/F案

### 4.1 共通生成API
- `spawn_shape(transport_pid, shape, opts)`  
  - `shape`: `:quad | :cube | :sphere | :cylinder | :capsule | :ring | :grid`
  - `opts`:
    - `:name`（必須）
    - `:parent_id`（省略時 `"Root"`）
    - `:position`（省略時 `%{x: 0, y: 1.4, z: 0.5}`）
    - `:scale`（省略時 `%{x: 0.5, y: 0.5, z: 0.5}`）
    - `:color`（省略時 `%{r: 1, g: 1, b: 1, a: 1}`）
  - 戻り値:
    - 成功: `{:ok, %{slot_id: String.t(), mesh_id: String.t(), material_id: String.t(), renderer_id: String.t()}}`
    - 失敗: `{:error, reason}`

### 4.2 図形別ショートカットAPI
- `spawn_quad(transport_pid, opts)`
- `spawn_cube(transport_pid, opts)`
- `spawn_sphere(transport_pid, opts)`
- `spawn_cylinder(transport_pid, opts)`
- `spawn_capsule(transport_pid, opts)`
- `spawn_ring(transport_pid, opts)`
- `spawn_grid(transport_pid, opts)`
- 戻り値は `spawn_shape/3` と同一

## 5. 内部仕様

### 5.1 componentType 対応表
- `:quad` -> `[FrooxEngine]FrooxEngine.QuadMesh`
- `:cube` -> `[FrooxEngine]FrooxEngine.BoxMesh`
- `:sphere` -> `[FrooxEngine]FrooxEngine.SphereMesh`
- `:cylinder` -> `[FrooxEngine]FrooxEngine.CylinderMesh`
- `:capsule` -> `[FrooxEngine]FrooxEngine.CapsuleMesh`
- `:ring` -> `[FrooxEngine]FrooxEngine.RingMesh`
- `:grid` -> `[FrooxEngine]FrooxEngine.GridMesh`

### 5.2 実行シーケンス
1. `addSlot`
2. `addComponent`（Mesh）
3. `addComponent`（`PBS_Metallic`）
4. `addComponent`（`MeshRenderer`）
5. `updateComponent`（`MeshRenderer.Materials`）
6. `updateComponent`（`PBS_Metallic.AlbedoColor`）

## 6. エラー方針
- 入力値不正（shape/opts不足）:
  - `{:error, :invalid_request}`
- 接続未確立/送信不可:
  - `{:error, :not_connected}` または `{:error, :transport_error}`
- Resonite 側失敗:
  - `{:error, {:remote_error, detail}}`

## 7. テスト方針
- 単体テスト:
  - shapeごとの `componentType` 変換
  - メッセージ組み立て結果の検証
  - 不正入力時の `:invalid_request`
- 実機確認:
  - `quad` `cube` `sphere` `cylinder` `capsule` `ring` `grid` の順に生成し、各応答成功を確認
