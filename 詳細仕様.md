# 詳細仕様

## 1. 通信方式
- 推奨: `WebSocket + JSON`
- 採用理由:
  - Elixirで実装しやすい
  - 双方向通信が可能
  - デバッグしやすい（テキスト可視）

## 2. Elixir実装詳細

### 2.1 モジュール分割
- `Resonite.Client`
  - WebSocket接続管理
  - 再接続（指数バックオフ）
  - 送受信ループ
- `Resonite.Protocol`
  - コマンドJSONエンコード
  - 応答JSONデコード
  - バリデーション
- `Resonite.Scene`
  - 高レベルAPI提供（spawn/update/delete/property変更）

### 2.2 想定公開API
- `Resonite.Client.start_link/1`
- `Resonite.Scene.spawn/2`
- `Resonite.Scene.set_transform/3`
- `Resonite.Scene.set_property/5`
- `Resonite.Scene.delete/2`

### 2.3 ライブラリ
- WebSocketクライアント: `WebSockex`

## 3. メッセージ仕様
- フォーマット: 1リクエスト1行JSON
- 共通フィールド:
  - `id`: リクエスト識別子（応答突合用）
  - `op`: 操作種別

### 3.1 spawn
`kind` に指定可能な値（初期対応）:
- `cube`
- `sphere`
- `cylinder`
- `capsule`
- `plane`
- `quad`
- `text`
- `empty`

```json
{"id":"req-1","op":"spawn","kind":"cube","name":"BoxA","transform":{"position":[0,1,0],"rotation":[0,0,0,1],"scale":[1,1,1]}}
```

### 3.2 set_transform
```json
{"id":"req-2","op":"set_transform","target":"BoxA","transform":{"position":[1,1,0],"rotation":[0,0,0,1],"scale":[1,1,1]}}
```

### 3.3 set_property
```json
{"id":"req-3","op":"set_property","target":"BoxA","component":"DynamicValueVariable`1[System.Single]","property":"Value","value":0.75}
```

### 3.4 delete
```json
{"id":"req-4","op":"delete","target":"BoxA"}
```

## 4. 応答仕様

### 4.1 成功
```json
{"id":"req-1","ok":true,"result":{"ref":"slot://..."}}
```

### 4.2 失敗
```json
{"id":"req-3","ok":false,"error":"component not found"}
```

### 4.3 エラー分類（推奨）
- `invalid_request`: 必須項目不足/型不一致
- `not_found`: target/componentが存在しない
- `conflict`: 同名生成など競合
- `internal_error`: 実行時例外

## 5. Resonite側実装要件
- 外部入力受信口（Mod/LogiX/ブリッジ）を用意する。
- 受信JSONを解釈し、対象オブジェクト操作へマッピングする。
- 応答で `id` を必ずエコーして返す。

## 6. 接続・信頼性
- 接続断時は自動再接続。
- 送信中断時は失敗を呼び出し元へ通知。
- タイムアウトを設け、未応答リクエストを解放。

## 7. 検証詳細
- 連続100回 `set_transform` 実行で欠落なし。
- 通信断後に自動復帰し再送可能。
- 不正JSON/不正`op`で `ok:false` を返す。
- 同名オブジェクト時の挙動（拒否 or 上書き）を仕様通り確認。

## 8. 今後の拡張
- メッセージに `version` を導入。
- 識別子を `name` からGUID中心へ移行。
- 認証トークンや許可操作制限を追加。
