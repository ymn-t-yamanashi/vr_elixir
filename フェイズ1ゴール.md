# スプリント1ゴール（確定版）

## 目的
Elixir から ResoniteLink に接続し、代表命令を安定実行できる最小実用基盤を完成させる

## 実装対象命名
- Mixプロジェクト名は `:resonite_link_ex` とする
- ルートモジュール名は `ResoniteLinkEx` とする

## 方針
- スプリント1は「全命令対応」ではなく「代表命令対応」とする
- 品質は Elixir ツールによるゲートで担保する
- 開発・実行は Docker 環境で統一する

## ゴール定義
- Docker 環境でセットアップから実行まで再現できる
- ResoniteLink 接続、切断検知、再接続ができる
- 以下の代表命令を Elixir から実行できる
  - `requestSessionData`
  - `addSlot`
  - `updateSlot`
  - `addComponent`
  - `updateComponent`
  - `removeComponent`
  - `removeSlot`
- サンプルとしてワールド内に四角形（Quad）を表示できる
- 送受信ログに `messageId`、`$type`、成功/失敗、エラー内容が残る

## 完了条件（DoD）
- `mix format --check-formatted` 成功
- `mix credo --strict` 成功
- `mix test` 成功
- E2E 手順で Slot 生成 → 更新 → Component 追加/更新/削除 → Slot 削除を再現
- E2E 手順で四角形（Quad）表示を再現
- 再接続試験で 3 回連続成功
- 失敗ケースを確認
  - 不正 `$type`
  - 存在しない `slotId` / `componentId`

## スコープ内
- Elixir クライアント基盤（接続、送受信、再接続）
- 代表命令 API
- 失敗時の標準エラーハンドリング
- 実行手順書と最小テスト

## スコープ外
- 全命令の完全実装
- 高度なアセット取り込み最適化
- 本番監視、メトリクス基盤、長期運用設計

## スプリント2への引き継ぎ
- 全命令対応へ拡張
- 命令ごとの自動生成や型安全 API の整備
- E2E の自動化拡張
