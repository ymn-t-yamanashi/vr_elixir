# PJ Rule Guard

## 利用タイミング
- `PJルール.md` または `Elixirルール.md` を編集したとき
- ルール文書の要約・圧縮・整理を行うとき

## 手順
1. 変更対象を `git diff -- PJルール.md Elixirルール.md` で確認する。
2. 以下の要素が維持されているか確認する。
- 条件値（例: `90%以上`, `100%`）
- 必須事項（例: Docker内実行、品質ゲート）
- 禁止事項（例: 壁打ち中の無断編集禁止）
3. 条件値や必須/禁止が変わる場合は、変更理由を明記してから反映する。
4. 不明な変更がある場合は、編集を止めて確認を取る。

## 出力方針
- 変更が安全: 「維持できている条件」を列挙する。
- 変更にリスク: 「変わった条件」を具体的に列挙する。
