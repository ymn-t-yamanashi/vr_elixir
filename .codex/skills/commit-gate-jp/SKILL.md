# Commit Gate JP

## 利用タイミング
- コミット直前
- 複数ファイル変更後の最終確認時

## 手順
1. `git status --short` で変更ファイルを確認する。
2. コミット対象外が混在していないか確認する。
3. 品質ゲートは `.codex/skills/elixir-quality-gate/SKILL.md` の手順で実行する。
4. コミットメッセージが日本語であることを確認する。
5. `git add` 対象を明示してコミットする。

## 失敗時の対応
- 品質ゲート失敗: 失敗ファイルのみ最小修正して再実行
- 取り残し検知: コミット対象を分割して再実行
- メッセージ違反: 日本語メッセージに修正

## 参照
- 品質基準値・必須条件・禁止事項は `Elixirルール.md` を正本とする。
