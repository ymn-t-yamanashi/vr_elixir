# 基本設計

## 1. 目的
ElixirアプリからResoniteワールド内オブジェクトを外部操作できるようにする。

## 2. 対象範囲
本システムで提供する操作は以下。
- オブジェクト生成
- Transform（位置・回転・スケール）変更
- コンポーネント値変更
- オブジェクト削除

## 3. システム構成
- Elixirアプリ
  - 外部制御APIを提供し、Resoniteへコマンド送信する。
- Resonite側受信層
  - 外部入力を受信し、ワールド操作に変換する。
- 通信路
  - ElixirとResonite間のコマンド/応答の伝送を担う。

## 4. 設計方針
- 送受信はリクエスト/レスポンス型のプロトコルで統一する。
- 操作コマンドは拡張可能な形式（操作種別を明示）にする。
- 接続断や不正コマンドを考慮したエラーハンドリングを行う。
- 段階的導入（移動操作 → 生成/削除 → 値更新）でリスクを下げる。

## 5. 非機能要件
- 接続断時に自動復旧できること。
- 連続更新時に取りこぼしを最小化すること。
- 失敗時に原因を判別可能な応答を返すこと。

## 6. 運用・拡張方針
- 初期は操作対象を限定し、安定後に対象オブジェクトや操作種別を拡張する。
- オブジェクト識別、認証、運用制約は環境に応じてポリシー化する。
- 詳細技術仕様は `詳細仕様.md` に定義する。

## 7. マイルストーン
1. 外部コマンドで既存オブジェクト移動を実現。
2. オブジェクト生成/削除を実装。
3. コンポーネント値更新を実装。
4. 再接続・エラーハンドリングを強化。
5. 負荷時の安定性を検証。
